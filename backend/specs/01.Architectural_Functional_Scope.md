## General

Budgeteer is an app for personal budget control. 

## Functionality

Budgeteer app must allow the consumer the following use cases:

- Register in the platform prividing personal information. 
- Login in the app to access the features available and information related to his budgets, incomes and expenses.
- Logout from the app
- Create a new Budget.
- Add a new transaction
- Access the dashboard page to see the details of its budget, incomes, expenses and transactions
- Access the insights page to see analytics about its budgets, consumer behaviors, detailed reports and recommendations based in best practices
- Access the projections page to see the projections until budget end date of its expenses and incomes based in historical data.
- Access the reporting page to generate xlsx and pdf reports available in app.
- Access the profile page to see/update profile and personal data

Budgeteer app must allow the administratios the following use cases:

- Register in the platform prividing personal information. 
- Login in the app to access the features available and information related to users and budgets
- Logout from the app


## Architecture

- It's a full stack application composed by Backend/API and Web Frontend.
- Frontend: Developed in react.js version 19 + vite 8.0 (stable version) + typescript 5.x (stable version) + tailwindcss 4.2
- Backend: Developed in node.js latest stable version  + express.js 5.1.0  + zod for input data validation for API REST 
- Database: Implemented in mongodb atlas cloud service with Prisma ORM v6.19
- Mobile app: It won't be implemented in the first version of app

## Layers

Backend app must have a layered architecture conformed by:

- Routes. Routes to several endpoints
- Controllers. handle requests and responses from/to services and routes including errors that will be managed by global error handler.
- Services. Implement the service logic and access to DB layer using prisma orm with properly exception handling
- Middleware. Implement several middlewares:
    1. Input validation using zod for exceptions and formating
    2. Global Error Handler including exceptions for authentication, authorization, database (prisma).
- DB (prisma schema)
- Utils. For password hashing and validation and jwt generation and validation
- Config. Under this folder must me managed the environment variables using a config file pattern.

## Restrictions in the first version

- Only one budget is allowed per user
- username and password is the only authentication method
- no admin functionality will be implemented in this phase
