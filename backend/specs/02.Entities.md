# Entities
The entities related to the project apply for prisma ORM and for the models required in the backend and frontend app.

## Person (model)
User has the following properties:
- id: Provided by database
- firstName: string. No more than 20 alphabetical characters.
- lastName: string. No more than 20 alphabetical characters.
- username: string. No more than 20 alphanumeric characters.
- password: string. No more than 20 alphabetical characters.
- createdAt: Datetime. date and time of creation 
- updatedAt: Datetime. date and time of updating

### User (inherit from Person)
User has the following properties:
- id: Provided by database
- firstName: string. No more than 20 alphabetical characters.
- lastName: string. No more than 20 alphabetical characters.
- username: string. No more than 20 alphanumeric characters.
- password: string. No more than 20 alphabetical characters.
- budgets: Budget[]
- transactions: Transaction[]
- createdAt: Datetime. date and time of creation 
- updatedAt: Datetime. date and time of updating

### Admin (inherit from Person)
Admin has the following fields:
- id: Provided by database
- firstName: string. No more than 20 alphabetical characters.
- lastName: string. No more than 20 alphabetical characters.
- username: string. No more than 20 alphanumeric characters.
- password: string. No more than 20 alphabetical characters.
- createdAt: Datetime. date and time of creation 
- updatedAt: Datetime. date and time of updating

## Budget (Model)
Budget has the following properties:
- id: Provided by database
- name: string. No more than 50 alphanumeric characters. Spaces allowed.
- user: User
- beginningDate: Datetime
- endingDate: Datetime
- incomes: BudgetCategory[] with type "incomes"
- expenses: BudgetCategory[] with type "expenses"
- createdAt: Datetime. date and time of creation 
- updatedAt: Datetime. date and time of updating

### BudgetCategory (Composite type)
- name: string. No more than 20 alphanumeric characters.
- type: "incomes" | "expenses"
- budgetGroups: budgetGroup[]


#### BudgetGroup (Composite type)
- name: string. No more than 20 alphanumeric characters.
- type: "incomes" | "expenses"
- budgetItems: BudgetItem[]


##### BudgetItem (Composite type)
- name: string. No more than 20 alphanumeric characters.
- plannedDate: Datetime
- plannedAmount: float
- type: "expense" or "income"
- currency: string
- frequency: ["daily" | "weekly" | "monthly" | "quarterly"  | "semiannually" | "annualy" | "onetime" ]
- buckets: ItemBucket[]

###### ItemBucket (Composite type)
- plannedDate: Datetime
- currentDate: Datetime
- plannedAmount: float
- currentAmount: float
- currency: string

## Transaction (Model)
User has the following properties:
- id: Provided by database
- txuser: User
- txdatetime: Datetime. date and time of creation
- txcurrency: string
- txamount: float
- txtype: "income" or "expense"
- txcategory: BudgetCategory name 
- txgroup: BudgetGroup name
- txitem: BudgetItem name depending if income or expense


The Budget model hierarchy is as follow:

- Budget has incomes and expenses. Incomes and expenses have several BudgetCategory's: 
- Example  of Budgetcategory for incomes: Salaries, Commisions, Dividends
- Example  of BudgetCategory for expenses: Utilities, Communities, Subscriptions
- A BudgeCategory contains several BudgetGroup's
- Example of BudgetGroup for Subscriptions: Software, Streaming, Magazines
- A BudgeGroup contains several BudgetItem's
- Example  of BudgetItem's for Streaming's BudgetGroup: Netflix, Spotify, HBO
- Each BudgetItem has at least one ItemBucket. For example, if Netflix it's a monthly subscription, and the budget has a duration of 1 year, then the Netflix's BudgetItem has 12 ItemBuckets.
- The ItemBuckets must be created by the budget service once the BudgetItem is defined.
- Each ItemBucket copy the info from its parent BudgetItem related to: plannedDate, plannedAmount, type and currency.
- Each transaction is allocated in the corresponding ItemBucket depending of transactio's item (txitem) and txdatetime vs BudgetItem name and plannedDate of ItemBucket.

### Rules for bucket creation
- For each BudgetItem, the number of ItemBuckets depend of the budget's beginning and ending dates and also the frequency of the budgetItem. 
- The first and last buckets should match only the fraction of days/weeks/months/quarters/halfs/years of the corresponding initial and last periods covered by the budget duration.

The following examples define the behaviour expected:
- For example, if the duration of the Budget is 1 year, and we create a BudgetItem with frequency "monthly", then 12 buckets should be created.
- If the duration of the Budget is 4 months, and we create a BudgetItem with frequency "daily", then the number of buckets should correspond to the number of calendar days (Mon-Sun) found in that period including first and last day of the budget period.
-  If the duration of the Budget is 6 months, and we create a BudgetItem with frequency "weekly", then the number of buckets should correspond to the number of calendar weeks (week start on Monday to Sunday) found in that period, depending of the real dates. 
-  If the duration of the Budget is 1 year, and we create a BudgetItem with frequency "monthly", then the number of buckets should correspond to the number of calendar months found in that period, depending of the real dates. 

### Rules for transactions matching to buckets

- The transaction field txitem defines in which BudgetItem should be inserted. 
- The transaction field txdatetime defines which ItemBucket should be inserted. The bucket allocated should be the one with plannedDate closest to the txdatetime.
